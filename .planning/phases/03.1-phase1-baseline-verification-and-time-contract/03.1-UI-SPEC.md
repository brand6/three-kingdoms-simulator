## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Tight inline separation inside compact menu/list rows |
| sm | 8px | List row gaps, submenu stacks, officer/task row spacing |
| md | 16px | Default panel padding, popup margins, card internal spacing |
| lg | 24px | Main content column gap, section separation |
| xl | 32px | Root scene margin, top-bar region separation |
| 2xl | 48px | Minimum interactive height for bottom-bar and confirm buttons |
| 3xl | 64px | Top-bar shell minimum height |

Exceptions: 12px is allowed for dense text stacks inside overview cards and popup content; keep 72px as BottomBar minimum shell height.

## Interaction Contract

- Keep the existing **single-scene MainScene overlay model**; do not introduce scene switches for verification work.
- Top-bar field order is fixed as: **Time → City → Identity → Faction → Clan/Family**.
- `TimeLabel` is an **always-visible value-only field**. Render the exact shared format from `TimeManager.get_current_label()`: `190年 / 1月 / 第1旬`. `MainHUD` must not prepend `时间：` or rebuild the year/month/xun string locally.
- Loading, success, and error states must preserve the **same top-bar shell and field positions** so cold-start verification can compare one stable surface.
- Left overview field order is fixed as: **Name → Portrait/State placeholder → AP → Energy → Stress → Fame → Merit → Office → Status → Health**.
- Bottom-bar order is fixed as: **行动 → 角色(禁用) → 关系 → 势力 → 家族/士族(禁用) → 日志(禁用) → 结束本旬**.
- Future-phase entries remain **visible but disabled**; do not replace them with placeholder scenes or hidden navigation.
- Modal behavior remains explicit and sequential: **End Xun confirmation → Xun Summary**, and **Month Report → Promotion Popup**. Never open the month-task picker while month-end feedback is active.
- Informational/result dialogs use a **single primary confirm action**; destructive flows require **explicit confirm + cancel**.
- Keep current modal shell sizes unless a regression fix requires otherwise: Action Result `560×360`, End Xun `420×180`, Xun Summary `620×420`, Month Report `620×420`, Promotion `520×320`, Faction Panel `760×520`.
