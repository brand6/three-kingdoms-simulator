# Phase 1 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

Source basis: `01-CONTEXT.md` D-04~D-06, `REQUIREMENTS.md` CORE-01/02, `ROADMAP.md` Phase 1, `design/UIUX/原型 UI 流程图 v1.md` §4, plus default token decisions for unspecified visual values.

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Godot built-in `Control` / `Container` / `PanelContainer` / `Button` |
| Icon library | none — use text labels and simple placeholder glyphs only |
| Font | Godot Default Sans（如需中文显示异常，再切换系统无衬线字体，不在本阶段引入自定义字体资产） |

### Phase Surface Contract

- **入口流**：启动后自动加载默认 190 烟雾样本，直接进入 `MainScene`；不出现剧本选择或人物选择 UI。
- **页面数**：本阶段只交付 **1 个可操作页面**：主 HUD 总览页。
- **根节点契约**：HUD 根必须为 `Control`，使用全屏 anchors preset（铺满父容器），不得用手写绝对定位拼整页。
- **布局结构**：
  - 顶部信息条：64px 高
  - 左侧人物总览栏：320px 宽
  - 右侧上下文栏：320px 宽
  - 底部主操作条：72px 高
  - 中央区：用于城市/样本总览与加载状态说明
- **视觉焦点**：进入主 HUD 后，第一注意点必须落在中央区的 `已进入 190 样本` 摘要卡；顶部时间/状态条作为持续参考层，左侧人物总览作为第二扫描层，不得让顶部条或禁用导航抢走首屏焦点。
- **容器实现契约**：
  - 页面主骨架：`MarginContainer` → `VBoxContainer`
  - 顶部信息条 / 中部主体 / 底部操作条：由 `VBoxContainer` 垂直分段
  - 中部主体区：`HBoxContainer`
  - 左栏 / 中央区 / 右栏：统一使用 `PanelContainer` 承载内容
- **尺寸行为契约**：
  - 左右侧栏通过 `custom_minimum_size.x = 320` 固定最小宽度
  - 中央区 `size_flags_horizontal` 必须使用扩展填充（expand + fill）
  - 顶部条与底部条不得跟随中部内容挤压变高
- **目标分辨率**：以 1600×900 为基准布局，1280×720 下不得信息重叠或按钮截断。
- **信息优先级**：时间、城市、身份、势力、官职、AP、精力、压力、名望、功绩必须首屏可见，不允许折叠到二级页签。
- **交互边界**：不实现地图优先探索、不实现多页面跳转、不实现占位页。

### Godot 4.6 UI Implementation Contract

- **统一主题**：Phase 1 必须使用 **单一 `Theme` 资源** 管理字体、字号、按钮态、面板色与分隔色；禁止在大量单个节点上做零散 local override。
- **主题范围**：Theme 至少覆盖 `Label`、`Button`、`PanelContainer` 三类控件。
- **按钮状态**：主操作按钮必须通过 Godot 的 `disabled` 状态进入禁用，不允许仅靠自定义脚本拦截点击来伪装禁用。
- **提示信息**：交互说明优先使用 `tooltip_text`；但对于禁用入口，不得只依赖 tooltip，还必须提供常驻说明文字或状态说明区。
- **焦点规则**：当前阶段仅允许真正可执行的主 CTA 获取初始焦点；未来入口若为禁用态，不要求可聚焦。
- **装饰控件**：纯展示用标签/占位图若覆盖按钮热区，应将 `mouse_filter` 设为忽略，避免阻断点击。

### Navigation Contract

- 底部一级入口固定显示：`行动`、`角色`、`关系`、`势力`、`家族/士族`、`日志`。
- 上述入口在 Phase 1 **全部可见但禁用**，不得跳转到空白页或占位页。
- 禁用态统一规则：
  - 使用 Godot `Button.disabled = true`
  - 文字与背景走 Theme 的 disabled 态，不手写逐个颜色
  - 透明度视觉目标约 55%，但以 Theme disabled 对比度可读为准
  - 悬停可显示 `tooltip_text`，同时在底部操作条上方保留一行常驻说明：`灰色入口将在后续阶段开放`
  - 不使用弹窗打断
- 主 HUD 自身不需要额外“主页”按钮；进入游戏即位于主 HUD。

### State Contract

- **加载中**：中央区显示样本名、默认主角名与文案 `正在加载 190 样本…`，禁止显示空白背景。
- **加载成功**：中央区显示“已进入 190 样本”摘要卡，并默认聚焦左侧人物总览。
- **加载失败**：保留完整 HUD 框架，但中央区替换为错误卡；不得进入半渲染状态。
- **无任务/无事件**：右侧快捷信息区显示空状态文案，不留空白块。

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | 图标与数字间距、标签内微间距 |
| sm | 8px | 紧凑状态行、按钮内文字与图标间距 |
| md | 16px | 默认卡片内边距、普通控件间距 |
| lg | 24px | 面板分区内边距、HUD 模块间距 |
| xl | 32px | 左右主栏与中央区之间的布局间距 |
| 2xl | 48px | 大块信息分组之间的断开距离 |
| 3xl | 64px | 页面级留白、顶部与主体的主分隔 |

Exceptions: 一级入口按钮最小点击高度 48px；状态摘要卡最小高度 48px；错误/空状态卡最小内边距 24px。

### Layout Rhythm Rules

- 同一信息卡内部只允许使用 **8px / 16px** 两档间距。
- HUD 三大列之间固定使用 **32px** 间隔，不因内容多少动态压缩到 24px 以下。
- 顶部信息条内的字段组之间固定 **24px**，字段标签与字段值之间固定 **8px**。
- 底部主操作条按钮左右内边距固定 **16px**，相邻按钮间距固定 **8px**。
- 所有摘要卡与侧栏卡必须设置 `custom_minimum_size.y`，避免因文本过少而塌陷成过薄条块。

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 600 | 1.4 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

### Typography Contract

- `Display` 仅用于剧本标题、主角姓名、核心数值标题，不可用于普通按钮。
- `Heading` 用于面板标题、分区标题、错误/空状态标题。
- `Label` 用于字段名、按钮文字、徽章、禁用态说明。
- `Body` 用于说明文案、状态值、空状态正文。
- 本阶段禁止使用第三种字重；只允许 **400** 与 **600**。
- Godot Theme 中的 `Button.font_size`、`Label.font_size` 应与本表一致，不允许节点级随意漂移。

## Interaction Contract

### Main HUD Information Contract

- 顶部信息条必须按以下顺序显示：`190年 / 月 / 旬` → `当前城市` → `当前身份` → `所属势力` → `当前官职`。
- 左侧人物总览必须按以下顺序显示：`姓名` → `头像占位` → `AP` → `精力` → `压力` → `名望` → `功绩`。
- 右侧快捷信息区必须只保留 3 个模块：`当前任务`、`最近事件`、`重要提示`。
- 中央区只承担“当前样本概览 / 加载状态 / 基础说明”职责，不塞入未来系统详情。
- 顶部信息条与左侧人物总览字段都应以 `Label + Value` 成对组织，优先交给 `HBoxContainer` / `VBoxContainer` 自动布局，而非手工 offset 微调。

### Click Depth Contract

- 启动后进入主 HUD 的路径为 **0 次手动点击**。
- 查看核心状态的路径为 **0 次手动点击**。
- 识别未来可扩展入口的路径为 **1 次视线扫描**，不依赖隐藏菜单。

### Feedback Contract

- 数值展示格式统一为：`标签 + 当前值`，不展示本阶段未实现的波动箭头。
- 若字段缺失，显示 `—`，不得显示 `null`、`0/0`、`N/A` 或代码字段名。
- 错误状态必须同时说明 **问题是什么** 与 **下一步做什么**。
- 禁用按钮不能只靠颜色区分，必须保留禁用文案或聚焦提示。
- 所有可交互控件若后续会开放，命名与视觉位置在本阶段就要稳定，避免 Phase 2 起重新换位。

### Focus And Input Contract

- 进入主 HUD 后，默认焦点落在唯一可执行主 CTA（若存在）；若无可执行 CTA，则不强制抢焦点。
- 可交互按钮需保留标准键盘/手柄接受逻辑，不在 `_gui_input` 内重写整套基础点击行为。
- 若后续支持键盘/手柄导航，底部按钮顺序必须与视觉顺序一致：`行动` → `角色` → `关系` → `势力` → `家族/士族` → `日志`。
