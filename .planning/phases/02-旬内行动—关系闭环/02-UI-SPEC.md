## Primary Screens And Focal Points

| Screen / State | Primary focal point | Secondary information | Contract |
|----------------|---------------------|-----------------------|----------|
| 主 HUD 空闲态 | 中部主内容区 | 左侧人物状态、右侧上下文、底部导航 | 玩家进入界面后先看“现在能做什么”，而不是先被装饰元素吸引。 |
| 行动菜单展开态 | 底部“行动”按钮上方的浮动行动面板 | 顶部状态条持续可见 | 浮层必须成为当前唯一主要操作焦点，但不得遮掉顶部时间与左侧核心状态。 |
| 目标选择弹窗 | 居中目标弹窗 | 弹窗外 HUD 降低存在感 | 目标选择是第三层操作，需通过模态聚焦避免误触。 |
| 行动结果弹窗 | 居中结果卡片 | 右侧最近事件区可同步摘要 | 结果先解释“发生了什么”，再解释“数值怎么变了”。 |
| 旬末结算态 | 居中旬末总结面板 | 顶部时间与底部导航弱化 | 这是本旬的总结焦点，视觉优先级高于普通行动结果。 |

**Visual hierarchy:**
- 第一优先级：当前决策或反馈容器（行动浮层 / 目标弹窗 / 结果卡 / 旬末总结）
- 第二优先级：顶部时间与左侧人物状态
- 第三优先级：右侧提示信息与未激活的底部入口
- Phase 2 不使用纯图标主操作；所有可点击主操作必须保留文本标签

## Interaction Contract

### 1. Main HUD
- 保持 Phase 1 的三段结构：顶部状态条 / 中部三栏 / 底部主操作区。
- 顶部时间、城市、身份、势力、官职始终可见。
- 左侧人物状态始终可见：姓名、AP、精力、压力、名望、功绩。

### 2. Action Menu
- 点击底部 `行动` 后，打开**向上展开的轻量浮动面板**。
- 浮层内部采用**左分类 + 右动作列表**的双栏结构，但尺寸保持轻量，不覆盖整个中部主区。
- 一级分类固定显示：成长、关系、政务、军事、家族。
- Phase 2 首批只实装 `训练`、`读书`、`休整`、`拜访`、`巡察` 五个闭环动作；其他分类可显示为空状态或未开放提示。
- 身份权限导致不可用的动作直接隐藏；条件不足导致不可执行的动作继续显示，但需灰显并展示原因。

### 3. Target Selection
- 有目标的行动（至少 `拜访`）在选中动作后打开**单独目标弹窗**。
- 目标弹窗必须显示：目标名、所属势力、所在城市、关系摘要、执行该动作的预估效果。
- 无目标动作（`训练`、`读书`、`休整`）不额外跳转页面，直接进入确认/执行流程。

### 4. Result Feedback
- 每次行动完成后都要弹出结果卡片。
- 结果卡片必须按顺序展示：行动名 → 成败结论 → 原因说明 → 数值变化 → 关系变化 / 新线索。
- 失败行动不得静默结束；至少要返回“为什么失败”与“是否产生了部分效果/信息变化”。

### 5. Xun-End Feedback
- `结束本旬` 需要二次确认，不允许误触直接推进时间。
- 旬末总结必须展示：本旬行动摘要、主要数值变化、关系变化摘要、是否触发新提示。
- 玩家关闭旬末总结后回到主 HUD，并能立即开始下一旬规划。

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | 行内标签与数值间最小分隔、微型状态点缀 |
| sm | 8px | 按钮组、浮层内部紧凑列表、底部导航按钮间距 |
| md | 16px | 面板内默认间距、当前主题默认内边距、普通卡片内容间隔 |
| lg | 24px | 顶部状态条项目间距、浮层左右栏间距、弹窗段落间隔 |
| xl | 32px | 主 HUD 三栏间距、页面外边距、主分区间距 |
| 2xl | 48px | 旬末结算与大型结果面板的主分区留白 |
| 3xl | 64px | 不在 Phase 2 常态界面中使用，仅预留给更大屏幕或后续全屏报告页 |

Exceptions: none

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 400 | 1.4 |
| Heading | 20px | 600 | 1.25 |
| Display | 28px | 600 | 1.15 |

**Type hierarchy rules:**
- `Display` 只用于关键标题或阶段性结果标题，如“行动完成”“旬末总结”。
- `Heading` 用于面板标题、弹窗标题、栏位标题。
- `Body` 用于说明文、结果解释、空状态正文。
- `Label` 用于状态条、按钮辅助说明、条件原因、标签说明。

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | 执行行动 |
| Empty state heading | 当前分类暂无可执行行动 |
| Empty state body | 请切换其他分类，或等待 AP、精力、地点与目标条件满足后再试。 |
| Error state | 行动执行失败：条件已变化。请重新检查 AP、精力、地点与目标后重试。 |
| Destructive confirmation | 结束本旬：确认后将立即结算本旬变化并推进到下一旬。 |

**Copy rules:**
- 按钮文案优先采用“动词 + 名词”结构，例如 `执行行动`、`选择目标`、`结束本旬`。
- 灰显原因文案必须简短直述，例如：`AP 不足`、`精力不足`、`当前地点不可执行`、`暂无可拜访对象`。
- 行动结果标题必须避免空泛词；使用 `行动完成`、`行动失败`、`目标不可达`、`本旬已结束` 等明确表述。
