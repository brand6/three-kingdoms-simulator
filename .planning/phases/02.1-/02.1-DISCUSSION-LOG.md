## 月度节奏接入

| Option | Description | Selected |
|--------|-------------|----------|
| 自动弹出并必须选择 | 进入新月后立刻弹出任务选择，先定本月主任务，再继续本月行动 | ✓ |
| 自动弹出但可稍后再选 | 先提醒有任务可领，玩家可先做别的事，再回来选 | |
| 手动从HUD进入 | 不自动打断，由玩家自己从任务区或按钮打开 | |

**User's choice:** 自动弹出并必须选择
**Notes:** 用户希望月度事务感明确，新月必须先定本月正式事务。

| Option | Description | Selected |
|--------|-------------|----------|
| 第3旬结束后自动触发 | 结束当月第3旬时，先出月末结算，再进入下月第1旬 | ✓ |
| 单独点结束本月 | 把结束本旬和结束本月分成两个操作 | |
| 月末先提醒再确认 | 第3旬结束时先提示“进入月末结算”，玩家再确认进入 | |

**User's choice:** 第3旬结束后自动触发
**Notes:** 用户接受在现有 xun-end 流程上叠加月末钩子，不再增加新操作层。

| Option | Description | Selected |
|--------|-------------|----------|
| 必须先选任务才能行动 | 月初先确定正式事务，再开始本月旬内行动 | ✓ |
| 可先行动但不给任务进度 | 允许先做事，但这些行动不计入主任务推进 | |
| 可先行动并补记进度 | 允许先做事，选任务后按条件追溯记部分进度 | |

**User's choice:** 必须先选任务才能行动
**Notes:** 用户不希望出现“先行动后补任务”的模糊状态。

| Option | Description | Selected |
|--------|-------------|----------|
| 不允许更换 | 选定 1 个主任务后整月锁定，到月末结算 | ✓ |
| 允许放弃并重选一次 | 每月可撤销一次，降低试错成本 | |
| 允许随时更换 | 更自由，但削弱月度事务责任感 | |

**User's choice:** 不允许更换
**Notes:** 用户要的是“责任制月任务”，不是任务池自由切换。

## 任务推进规则

| Option | Description | Selected |
|--------|-------------|----------|
| 复用现有行动做推进 | 不新增任务专属行动；任务只读取现有行动结果与次数来累计进度 | ✓ |
| 现有行动为主+少量专属判定 | 仍复用现有行动，但个别任务允许额外条件 | |
| 新增任务专属行动 | 为任务单独加新行动入口 | |

**User's choice:** 复用现有行动做推进
**Notes:** 用户明确不希望为 2.1 另起任务专属玩法层。

| Option | Description | Selected |
|--------|-------------|----------|
| 统一进度值阈值 | 所有任务都汇总成 current/target/bonus 三段式进度 | ✓ |
| 按条件清单完成 | 每个任务列出2-3个完成条件，逐项勾选 | |
| 任务类型各自一套 | 各任务分别结算 | |

**User's choice:** 统一进度值阈值
**Notes:** 用户倾向最稳的数据驱动方案。

| Option | Description | Selected |
|--------|-------------|----------|
| 达到 bonus 阈值即可 | 统一用 bonus_value 判定优秀完成 | ✓ |
| 核心成功+特定附加条件 | 先成功，再满足附加条件才算优秀 | |
| 由月末评价人工加权 | 月末根据多项表现综合决定是否优秀 | |

**User's choice:** 达到 bonus 阈值即可
**Notes:** 用户希望优秀完成保持统一规则，不做人工加权。

| Option | Description | Selected |
|--------|-------------|----------|
| 当月未达成功阈值即失败 | 简单直接，最利于原型验证 | ✓ |
| 未达标算未完成，失败只给少量惩罚 | 语义更柔和，减少挫败 | |
| 按任务类型区分失败与未完成 | 有些任务失败，有些只是不够优秀 | |

**User's choice:** 当月未达成功阈值即失败
**Notes:** 用户接受 2.1 的任务结果层级先保持简单。

## 任务与月报界面

| Option | Description | Selected |
|--------|-------------|----------|
| 名称+发布人+描述+奖励 | 月初任务卡最小必要信息 | ✓ |
| 再加难度和推荐方向 | 信息更完整，但卡片更重 | |
| 再加成功/优秀条件摘要 | 选择时就明确阈值，但首屏更重 | |

**User's choice:** 名称+发布人+描述+奖励
**Notes:** 用户要任务选择足够快读，不想先把任务卡做成规则表。

| Option | Description | Selected |
|--------|-------------|----------|
| 当前主任务+进度+剩余旬数 | 最能支撑玩家每旬决策 | ✓ |
| 当前主任务+本月目标文案 | 更轻，但推进感较弱 | |
| 当前主任务+最近推进记录 | 反馈感强，但不如进度/剩余时间直接 | |

**User's choice:** 当前主任务+进度+剩余旬数
**Notes:** 用户希望 HUD 任务区是决策辅助，而不是只做叙事提示。

| Option | Description | Selected |
|--------|-------------|----------|
| 单屏总览报告 | 一屏展示任务结果、数值变化、是否升官、下一级目标 | |
| 先任务结果再升官弹窗 | 拆成两段反馈，更有仪式感 | ✓ |
| 分页报告 | 信息最完整，但原型阶段偏重 | |

**User's choice:** 先任务结果再升官弹窗
**Notes:** 用户要把“任务结果”和“任命/升官”明确分成两次反馈。

| Option | Description | Selected |
|--------|-------------|----------|
| 显示距离下一级还差多少功绩 | 最直观 | ✓ |
| 显示失败原因短句 | 解释为什么暂不具备升迁资格 | |
| 两者都显示 | 差值 + 挡住原因 | |

**User's choice:** 显示距离下一级还差多少功绩
**Notes:** 在后续讨论中，这一项被扩展为“显示距离目标官职还差多少关键数值”，不再局限于功绩单值。

## 升官反馈与影响范围

| Option | Description | Selected |
|--------|-------------|----------|
| 任务池层级+身份文本 | 升官后能接更高阶任务，且月报/HUD/称呼文本变化 | ✓ |
| 再加HUD官职目标提示 | 再在主界面强调“当前官职/下一目标” | |
| 再加可执行权限变化 | 直接开放新权限或新面板 | |

**User's choice:** 任务池层级+身份文本
**Notes:** 用户不希望 2.1 直接滑向 Phase 3 的权限政治。

| Option | Description | Selected |
|--------|-------------|----------|
| 独立升官弹窗 | 先看任务结果，再单独弹出升官反馈 | ✓ |
| 月报内一块区域直接显示 | 所有信息留在同一份月报里 | |
| 只改HUD不弹窗 | 最轻，但可见性较弱 | |

**User's choice:** 独立升官弹窗
**Notes:** 与上面的两段式月报结构一致。

| Option | Description | Selected |
|--------|-------------|----------|
| 要，明确提示新任务层级已解锁 | 让玩家立刻理解升官会影响下月事务 | |
| 只在下月任务选择时体现 | 不提前解释，让玩家下月自然发现变化 | ✓ |
| 只在月报里用一句模糊文案带过 | 只做轻提示 | |

**User's choice:** 只在下月任务选择时体现
**Notes:** 用户不要求当月反馈把任务池变化解释到位。

| Option | Description | Selected |
|--------|-------------|----------|
| 显示简短目标提示 | 如“距主簿级还差 7 功绩” | |
| 只在月报里显示 | 平时不占 HUD 空间 | |
| 不显示 | 让玩家只通过结果反馈理解成长 | ✓ |

**User's choice:** 用户以自由文本修正该题
**Notes:** 用户明确指出“升官不是固定路线”，并改为：升官必须满足 1）目标官职有空缺；2）功绩或声望达到官职要求；3）收到君主或太守等的任命通知。随后进一步确认：Phase 2.1 只做单候选人、单上级通知、空缺校验 + 达标校验的简化任命版；官职要求采用 **数据表配置**；任命通知来源采用 **按官职归属配置发送人**；主界面平时只显示当前官职，不显示候补目标提示。
