## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline delta chips, compact label-to-value gaps |
| sm | 8px | Button gaps, selector row spacing, compact metadata stacks |
| md | 16px | Default panel padding, task card padding, dialog inner margins |
| lg | 24px | Task card section spacing, main monthly report section gaps |
| xl | 32px | Scene edge margins, top bar spacing, major three-column layout gaps |
| 2xl | 48px | Minimum action button height, major dialog breathing room |
| 3xl | 64px | Top-level popup/report vertical rhythm only |

Exceptions: none. Dense stacked content inside task cards, month report sections, and selector/profile popups must use `sm` (8px) for compact stacking and `md` (16px) for readable section separation; any persistent HUD bar or dialog action row touched in this phase must normalize to `3xl`-adjacent layout rhythm via 64px container height or 48px button height only.

Source notes: existing `MainScene.tscn` already uses 32px outer margins, 24px main content gap, 16px card gaps, 8px button gaps, and 48px button heights; this spec preserves that rhythm for month-start and month-end overlays.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #F5F0E6 | Full-scene background, dialog canvas, non-interactive report surfaces |
| Secondary (30%) | #E4D8C5 | Cards, sidebar panels, popup panels, task summary rail, neutral buttons |
| Accent (10%) | #2F5D62 | Primary CTA, selected task card border, current task progress, promotion success header, key positive delta badges |

Accent reserved for: `领取主任务` primary button, currently selected monthly task card, task progress number or progress bar, promotion success title, and positive month-end summary badges. Do not use accent on every nav button or every clickable row.

Source notes: colors come from `PrototypeTheme.tres` (`PageBackground`, `PanelSurface`, `Accent`, `Destructive`). Phase 02.1 context adds accent-worthy moments at task lock-in, visible progress, and promotion outcome.

## Visual Hierarchy

- **Month-start task picker focal point:** the selected task card is the first visual priority. Use accent border/state on exactly one selected card, place `领取主任务` as the single primary action at the bottom-right of the dialog, and keep all non-selected cards in secondary surface treatment.
- **Month-start reading order:** dialog title → short institutional explanation (`本月尚未领受公事，请先择定一项主任务`) → task card list → selected-task reward summary → `领取主任务` CTA. Do not place flavor text above the selectable task list.
- **Month-start card hierarchy:** each task card shows task name first, issuer second, one short description block third, expected reward last. Hidden details such as thresholds or formulas must not compete with the primary card scan.
- **Month-end feedback focal point:** always present feedback in two steps: first the task-result month report, then the promotion dialog. The task-result report owns causal explanation; the promotion dialog owns outcome confirmation.
- **Month-end reading order:** task name → success / failure verdict → current progress versus threshold → key merit / fame / trust changes → one political-meaning summary line. If promotion fails, the standardized blocker label is the final emphasis before close.
- **Promotion dialog hierarchy:** office title or `未获任命` verdict first, issuer second, concise reason third. Use accent only for successful appointment; use destructive only for failed thresholds or hard blockers.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | 领取主任务 |
| Empty state heading | 本月暂无可领任务 |
| Empty state body | 当前官职暂无匹配事务。请检查任务池配置，或等待下月任务刷新后再作部署。 |
| Error state | 月度事务初始化失败。请检查任务模板、官职空缺、默认主角补丁与任命规则数据，然后重新启动项目。 |
| Month-start gate copy | 本月尚未领受公事，请先择定一项主任务。 |
| Destructive confirmation | 锁定主任务：确认领取后，本月主任务将锁定，不能更换。是否继续？；结束本旬：确认后将立即结算本旬变化并推进到下一旬。 |
| Promotion failure labels | 功绩不足；名望不足；无空缺；任务未达标 |

Source notes: `领取主任务` and task lock wording come from 02.1 context D-01/D-03/D-04/D-04a; month report field restraint and promotion failure labels come from D-11a/D-12; end-xun confirmation follows existing `MainHUD.gd` copy style; empty and error text are prescriptive defaults for this phase because upstream artifacts defined behavior but not exact strings.
