## 任务来源信息密度

| Option | Description | Selected |
|--------|-------------|----------|
| 中密度：来源+风险收益 | 保留现有任务名/请求方/描述/奖励，并新增：来源类型、关联人物或派系、潜在政治收益、潜在政治风险。信息够用，又不会把任务卡挤成数据表。 | ✓ |
| 低密度：只补来源 | 只在现有卡片上补来源类型和请求方，把政治收益/风险留到选中后的详情区。实现最稳，但首屏政治味会弱。 | |
| 高密度：首屏全展示 | 首屏同时展示来源类型、请求方、关联派系、收益、风险、推荐/反对提示。信息最完整，但很容易压缩可读性。 | |

**User's choice:** 中密度：来源+风险收益
**Notes:** 对“政治收益 / 风险”的具体表达，用户选择**标签词组**风格，而不是叙事短句或数值。

## 势力与人物政治摘要

| Option | Description | Selected |
|--------|-------------|----------|
| HUD只放3项，详情展开 | MainHUD 常驻只放最关键的 3 项：主要推荐人、主要阻力、当前机会/短板；更完整的派系态度和职位机会放到人物/势力面板。这样最稳，不会把主界面挤爆。 | ✓ |
| HUD放完整摘要 | MainHUD 直接放推荐人、阻力、派系态度、资格提示、机会摘要的大部分内容。主界面信息最全，但容易让中部卡片过重。 | |
| HUD只放一句总评 | MainHUD 只显示一句政治处境总评，把细节都交给人物/势力页。界面最简洁，但“政治始终在身边”的感觉会弱。 | |

**User's choice:** HUD只放3项，详情展开
**Notes:** 势力政治总览页第一页进一步确认：优先强调**派系块 + 玩家位置**，而不是资源表或单纯高层名单。

## 官职后果与动作权限

| Option | Description | Selected |
|--------|-------------|----------|
| 任务+信息可见性 | 升官后能看到更核心的任务来源、更多政治摘要和势力页信息；这是最容易被玩家立刻感知、也最不容易越界成全局操盘的后果组。 | |
| 发言资格+推荐能力 | 升官后开放进言/举荐类动作，并提升你对他人的政治影响力。政治味更强，但会牵动更多新行动与系统接口。 | |
| 失败代价+待遇变化 | 优先做高官失败后更重的信任下滑、机会变差、NPC称呼变化。氛围强，但正向“升官有何用”会稍弱。 | |
| 其他（用户自定义） | 官职变化应模拟该官职的历史定位，优先体现在任务类型解锁与少量额外行动，而不是只改信息展示。 | ✓ |

**User's choice:** 其他（用户自定义）
**Notes:** 用户补充：Phase 3 首批只做**少量代表性官职专属动作**；行动配置表新增**官职限定**列，留空表示全部官职都可用。用户给出“监察型官职可监察其他官员”作为设计参考风格。
