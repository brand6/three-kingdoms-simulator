## Resolution

root_cause: TaskSelectPanel still implements monthly task cards as a single plain-text Button template from the earlier UAT contract. _card_text() hard-codes the old field order and labels (including a separate “来源类型” line, “关联派系”, and “政治标签” with “机会:”/“风险:” prefixes), while _render_cards() renders that string directly into button.text. Because the card is not composed from structured sub-controls or rich text, the requested same-line layout and per-tag color distinction are impossible in the current renderer. The regression test phase21_monthly_hud_regression.gd also asserts the old labels, so the outdated layout is locked in by test coverage.
fix: Diagnose only. Update the card renderer to use structured inline UI or rich text, rename the displayed fields to the new UAT wording, and update the regression to assert the revised contract while preserving disabled confirm behavior before selection.
verification: Root cause confirmed by reading TaskSelectPanel.gd and phase21_monthly_hud_regression.gd; no alternate rendering path exists for task cards.
