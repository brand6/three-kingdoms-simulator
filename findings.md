# 发现与决策

## 需求
- 使用 Godot 引擎开发。
- 游戏基础体验参考《三国志10》。
- 融入《十字军之王》的一部分特性，而非纯三国志式体验。
- 项目核心应保持“单角色扮演、历史模拟、势力博弈”导向。
- 首个主剧本确定为 190 年剧本。
- 玩家可扮演全身份历史人物。
- 重点融合 CK 风格的士族门阀、联姻/子嗣、性格/关系、派系系统。
- 当前目标是先完成《项目总设计方案》。

## 研究发现
- 《三国志10》强项在于单武将视角、城市移动、任务驱动成长、舌战/单挑、仕官与自立。
- 《十字军之王》可借鉴的高价值特性通常包括：家族系统、婚姻/继承、性格与人际、封臣关系、派系与忠诚。
- 对 Godot 中小团队项目而言，最适合融合的是“关系与家族深度”，而非 CK 全量地图/封建契约仿真。
- 三国 190 剧本非常适合加入士族门阀：名门望族可影响仕途、婚配、举荐、忠诚、派系结构。
- “全身份”不应等于“全局策略游戏”，仍需坚持单角色约束，避免演变为纯君主 4X。
- 士族门阀系统可作为三国题材区别于普通武将 RPG 的核心特色层。

## 技术决策
| 决策 | 理由 |
|------|------|
| 优先考虑“单角色 + 势力沙盘”架构 | 符合用户目标，也更适配《三国志10》基底 |
| CK 特性采用模块化裁剪方案 | 降低系统耦合，便于原型优先开发 |
| 将 CK 的“家族”本地化为“士族/门阀 + 家族”双层结构 | 更贴合东汉末年至三国政治生态 |
| 将派系系统嵌入势力内部权力斗争，而非独立大地图玩法 | 能保留 CK 味道，同时不破坏单角色体验 |
